ERPNext AI Chatbot (Live in Production)

Challenge

Providing a seamless, conversational interface for ERP users to query complex business data without needing technical expertise or navigating multiple Frappe document views.

Architecture

Built a production-grade AI Chatbot leveraging the Model Context Protocol (MCP). The user interacts via a web UI, which routes queries to a backend MCP server. The server acts as a Tool Router, securely translating LLM intents into ERPNext API calls (with over 120+ tool categories supported).

WeldDB — Welding Data Management Platform

Challenge

Keepsake Automation's KUKA welding robots produced rich operational data — weld times, arc voltage, wire feed speed — but it was trapped in proprietary machine logs. QA reviews took hours; tracing defects to specific weld parameters was manual and error-prone.

Architecture

Built a pipeline that reads machine exports via FTP → Lambda ingestion → S3 raw storage → MySQL normalised store → React dashboard with live WebSocket updates. AWS CloudWatch handles alerting on anomalous readings.

AI Assistant for ERPNext with MCP Integration

Challenge

Non-technical ERP users needed live business data — sales figures, stock levels, pending tasks — but couldn't write Frappe filters or navigate document trees. Creating new reports required developer time every time.

Architecture

Built an MCP server exposing ERPNext as typed tools (get_doc, run_report, create_doc). Integrated with OpenAI via tool-calling so the AI can fetch live data, perform actions, and respond conversationally. Deployed behind Frappe's auth layer.

Technologies
PythonERPNext FrappeOpenAI API MCP ProtocolNode.js REST API
Results
60% reduction in report generation time
Non-technical staff can now self-serve data queries
AI can create and update ERP records via natural language
Secure — respects Frappe's role-based permissions

Real-Time Welding Analytics Dashboard

Challenge

Production managers had no centralised view of shop floor activity. Shift handovers relied on verbal briefings; there was no historical trend data to spot process degradation over time.

Architecture

Lambda ingestion functions publish to an SNS topic. S3 stores raw & aggregated time-series. A React dashboard polls a lightweight REST API (Node/Express on EC2) and renders live charts. CloudWatch dashboards feed the alerting layer.
